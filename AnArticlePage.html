The benefits of Client-side and server-side scripting

Both of them are useful in their own way.

Client-side scripting is allowed multiple interactive interfaces from the user. It doesn't require connection to the server. It can be more browser supportive. It can give more freehand or control for web widgets.

Server-side scripts users can create a template to the whole website. It is loading more faster than client-side scripting. Not required to download plugins such as Java, Flash. More safety so users only can inspect the HTML scripts.

Authentication and web security

Authentication is to confirming the user or person with a valid information such as, password, email orusername to gain access to the servers, files etc.

Web security is for prevent any secure risks between the user and the server. It gives more safe if the server is always up to date.

Session management

It is a secured process of multiple requests from the same user to the service. A session initialised by authenticating a user with a password, username, email address etc.

Session management is also a process for securing multiple requests to a service from the same user or entity.

So session management basically, is an authenticator for users to keep their accounts and information in safe.

Stateless programming

Stateless Programming is a way of developing app with state independent, usually functional programming languages.

There is no saved data about the communications (including previous datas) has to be handled based entirely on information.

Workplace Health and Safety (WHS)

Work Health and Safety Act 2011
A professional in an ICT industry must follow and accomplish the WHS act during the work hours, for example: cables on the ground must be take away from the walk path in the office, so the walk path must be free.
It will provide a safe and healthy environment for the co-workers and the staff of the business. It is mandatory and responsibility for everyone to make sure that everybody has a safe workplace.
